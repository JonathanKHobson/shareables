What It Is

The Shareables MCP is the shipping lane for small public HTML artifacts.

It keeps the manifest, local archive, style guide, structured renderer, validation checks, and GitHub Pages staging path in one place.

How It Works

  1. Create a manifest with title, audience, purpose, source paths, and listed or unlisted routing.
  2. Send a structured spec or simple Markdown to the MCP.
  3. Let the MCP render the styled HTML, validate it, stage it, and return the publish handoff.

What The MCP Owns

Style

The house tokens, shell, cards, buttons, focus states, and neutral Shareables mark.

Layout

Simple pages stay simple; larger pages get contents lists and tabs inside one HTML file.

Safety

Unlisted output gets noindex and stays out of public indexes, but it is still public by URL.

Safety Note

Unlisted does not mean private. Anything staged to GitHub Pages must be safe for public access by URL or repository browsing.

Content Size Rules

Small Content

Short shareables stay as one readable scrolling page.

Medium Content

Four or more sections, or roughly 800 words, add a contents list.

Large Content

Longer material stays in one file but gets accessible tabs and anchors.
